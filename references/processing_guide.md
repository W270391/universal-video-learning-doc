# 算法视频处理指南

## 概述

本文档提供算法视频文档生成器的详细使用指南和最佳实践。

## 支持的视频格式

### 推荐格式
- **容器格式**: MP4, MKV, AVI, MOV
- **视频编码**: H.264, H.265, VP9
- **音频编码**: AAC, MP3, WAV
- **分辨率**: 720p 或更高（代码识别需要清晰画面）

### 文件大小限制
- **建议最大**: 4GB
- **超过限制**: 可能需要分段处理

## 处理流程详解

### 阶段1: 预处理 (2-3分钟)
1. **视频分析**
   - 提取元数据（时长、编码、分辨率）
   - 检测音频流和视频流
   - 评估处理复杂度

2. **音频提取**
   - 转换为16kHz单声道WAV（Whisper推荐）
   - 应用降噪预处理（可选）
   - 分割长音频（>1小时）

### 阶段2: 语音识别 (20-40分钟)
1. **模型选择**
   - `tiny`: 最快，准确性较低
   - `base`: 平衡速度和准确性（推荐）
   - `small`: 更准确，速度较慢
   - `large`: 最准确，需要GPU加速

2. **语言设置**
   - 中文: `language='zh'`
   - 英文: `language='en'`
   - 自动检测: `language=None`

3. **时间戳生成**
   - 逐句时间戳（精确到秒）
   - 段落分组（按主题）

### 阶段3: 代码识别 (10-20分钟，可选)
1. **关键帧提取**
   - 每30秒采样一帧
   - 优先提取包含代码的帧
   - 过滤重复帧

2. **OCR处理**
   - 应用图像预处理（二值化、去噪）
   - 使用Tesseract识别文本
   - 过滤非代码内容

3. **代码结构化**
   - 识别代码块边界
   - 保持缩进和格式
   - 关联时间戳

### 阶段4: AI分析 (10-15分钟)
1. **内容理解**
   - 提取算法名称和类型
   - 识别核心概念
   - 分析解题思路

2. **结构化生成**
   - 按模板组织内容
   - 添加时间戳索引
   - 生成代码注释

3. **质量检查**
   - 验证完整性
   - 检查格式规范
   - 优化可读性

### 阶段5: 资源整合 (1-2分钟)
1. **洛谷题目搜索**
   - 提取算法关键词
   - 搜索相关题目
   - 筛选难度级别

2. **文档生成**
   - 合并所有内容
   - 生成Markdown
   - 添加目录和索引

## 性能优化

### 硬件建议
- **CPU**: 4核以上（推荐8核）
- **内存**: 8GB以上（推荐16GB）
- **GPU**: 可选（加速Whisper处理）
- **存储**: 10GB可用空间

### 处理优化
1. **并行处理**: 音频转录和代码识别可并行
2. **增量处理**: 长视频分段处理
3. **缓存机制**: 避免重复处理
4. **跳过OCR**: 无代码视频禁用OCR

## 故障排除

### 常见问题

#### 1. FFmpeg未找到
**症状**: `ffmpeg: command not found`
**解决**: 
- Windows: 添加FFmpeg到PATH
- macOS: `brew install ffmpeg`
- Linux: `sudo apt install ffmpeg`

#### 2. Whisper模型下载失败
**症状**: 模型下载超时或失败
**解决**:
- 使用国内镜像源
- 手动下载模型文件
- 使用较小的模型（base/tiny）

#### 3. OCR识别不准确
**症状**: 代码识别错误率高
**解决**:
- 调整图像预处理参数
- 使用更高分辨率的视频
- 手动校正识别结果

#### 4. 内存不足
**症状**: 处理过程中程序崩溃
**解决**:
- 分段处理视频
- 使用较小的模型
- 增加系统内存

## 高级功能

### 自定义模板
用户可以修改 `references/template.md` 来自定义输出模板。

### 批量处理
支持批量处理多个视频文件：
```bash
python algorithm_video_processor.py *.mp4 -o ./output/
```
